# Implementation Checklist

## 使用时机

实现过程中和实现完成后执行。

## 必查项

- [ ] 是否只实现 tasks.md 中的任务。
- [ ] 是否避免了未定义业务行为。
- [ ] 是否遇到未定义行为并停止反馈。
- [ ] 是否保持原流程行为。
- [ ] 是否遵守异常处理策略。
- [ ] 是否遵守幂等、重试、事务边界。
- [ ] 是否遵守日志和监控要求。
- [ ] 是否更新任务状态。
- [ ] 是否记录未完成项。
- [ ] 是否执行编译或基础测试。

## 阻塞条件

- 自行补业务规则。
- 未定义行为被直接实现。
- 编译失败。
- 核心测试失败。
- 实现与 Specification 冲突。
