# Task Checklist

## 使用时机

tasks.md 生成后、Implementation 前执行。

## 必查项

- [ ] 每个任务有唯一 ID。
- [ ] 每个任务有明确文件、模块或产物范围。
- [ ] 每个任务能追溯到 Specification 或 Plan。
- [ ] 不存在凭空新增业务任务。
- [ ] 任务顺序符合依赖关系。
- [ ] 包含核心开发任务。
- [ ] 包含核心测试任务。
- [ ] 包含必要文档或配置任务。
- [ ] 包含监控、日志或回滚任务。
- [ ] 每个任务可执行、可验证。

## 阻塞条件

- 核心需求没有任务覆盖。
- 核心测试缺失。
- 存在必须先澄清才能执行的任务。
- 任务要求实现未在 Specification 中定义的业务行为。
