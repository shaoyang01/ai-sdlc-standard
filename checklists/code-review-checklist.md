# Code Review Checklist

## 使用时机

代码实现完成后、Fix 或 Test 前执行。

## 必查项

- [ ] 是否符合 Specification 的行为约束。
- [ ] 是否保持原流程。
- [ ] 是否符合失败策略。
- [ ] 是否符合超时策略。
- [ ] 是否符合异常传播策略。
- [ ] 是否符合幂等策略。
- [ ] 是否符合事务边界。
- [ ] 是否存在数据一致性风险。
- [ ] 是否存在兼容性风险。
- [ ] 是否存在性能风险。
- [ ] 是否存在安全风险。
- [ ] 是否覆盖关键测试场景。
- [ ] Review 问题是否包含文件位置和修复建议。

## 阻塞条件

- 代码违反行为约束。
- 代码改变原流程但方案未允许。
- 异常处理与方案不一致。
- 状态流转错误。
- 数据写入错误。
- 存在核心测试失败风险。
