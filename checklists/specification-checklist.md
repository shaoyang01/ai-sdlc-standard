# Specification Checklist

## 使用时机

Specification Writing 完成后、Planning 前执行。

## 必查项

- [ ] 是否定义业务背景和目标。
- [ ] 是否明确 In Scope / Out of Scope。
- [ ] 是否描述原流程。
- [ ] 是否描述新流程。
- [ ] 是否明确条件未命中时保持原流程。
- [ ] 是否明确新逻辑失败时是否阻断原流程。
- [ ] 是否明确超时策略。
- [ ] 是否明确异常传播策略。
- [ ] 是否明确重试策略。
- [ ] 是否明确幂等策略。
- [ ] 是否明确事务边界。
- [ ] 是否明确状态流转。
- [ ] 是否明确数据来源和为空处理。
- [ ] 是否明确 DB 写入影响。
- [ ] 是否明确缓存影响。
- [ ] 是否明确 MQ 影响。
- [ ] 是否明确日志和定位字段。
- [ ] 是否明确监控和告警。
- [ ] 是否明确边界条件。
- [ ] 是否明确测试方案。
- [ ] 是否明确验收标准。
- [ ] 是否列出风险和残余风险。

## 阻塞条件

任一项影响核心行为且未定义，应标记为 Critical 或 High。
