# Sync Skill Contract

## 定义

Sync Skill 用于将稳定事实沉淀到知识库、Checklist、Schema、Workflow 或 Coding Guide。

## 必须遵守

- 只沉淀稳定、可复用事实。
- 不写入临时调试信息。
- 不覆盖短期规格源。
- 测试发现的规格遗漏必须回写 Checklist 或 Schema。
- 写入前必须说明目标和边界。

## 输入

- Specification
- Implementation Summary
- Review Report
- Test Feedback Report
- Existing Knowledge Base

## 输出

- Knowledge Updates
- Checklist Updates
- Schema Updates
- Sync Report

## 副作用

可能修改知识库或标准资产，必须明确授权和写入范围。

## 阻塞条件

- 事实未验证稳定。
- 写入目标不明确。
- 用户未授权写入。
- 规则只适用于当前一次临时需求。
