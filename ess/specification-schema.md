# ESS Specification Schema

## 适用范围

所有技术方案、设计方案、需求实现方案都必须遵循本 Schema。HTML、Markdown、PDF 只是渲染格式，不得改变 Schema 语义。

## 必填章节

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 背景 | 是 | 业务背景和问题来源 |
| 目标 | 是 | 本次要达成的业务目标 |
| Scope | 是 | In Scope / Out of Scope |
| 原流程 | 是 | 当前流程、状态、数据和调用链 |
| 新流程 | 是 | 修改后的流程、触发条件和分支 |
| 行为约束 | 是 | 哪些行为不能改变 |
| 实现约束 | 是 | 开发必须遵守的规则 |
| 状态流转 | 是 | 所有新增、修改或保持不变的状态变化 |
| 数据来源 | 是 | 数据来源、可靠性、为空时处理 |
| 数据变更 | 条件必填 | 有新增或修改数据时必填 |
| 接口变更 | 条件必填 | API/RPC/消息契约变化时必填 |
| 数据库变更 | 条件必填 | 表结构、索引、字段、DDL 变化时必填 |
| 缓存影响 | 条件必填 | Redis、本地缓存、缓存失效策略变化时必填 |
| MQ 影响 | 条件必填 | 生产、消费、重试、幂等变化时必填 |
| 日志 | 是 | 新增日志、关键字段、定位方式 |
| 监控 | 是 | 指标、告警、灰度观察点 |
| 异常处理 | 是 | 失败、超时、重试、降级、异常传播 |
| 边界条件 | 是 | 空数据、重复执行、并发、历史数据、兼容旧流程 |
| 测试方案 | 是 | 主路径、未命中路径、失败降级、兼容性验证 |
| 风险 | 是 | 已知风险、接受条件、残余风险 |
| 待确认事项 | 否 | 需要用户或业务方确认的问题 |

## 行为约束必须回答

- 条件未命中时是否保持原流程。
- 新逻辑失败时是否影响原流程。
- 新逻辑超时时是否影响原流程。
- 新逻辑异常是否允许向上传播。
- 是否改变原返回值。
- 是否改变原状态。
- 是否改变原事务边界。
- 是否改变原日志、MQ、缓存、DB 写入。

## 异常处理必须覆盖

- 接口失败
- RPC 失败
- MQ 失败
- DB 失败
- Redis 失败
- Timeout
- Exception
- 重复执行
- 并发冲突
- 数据为空

## 可测试性必须覆盖

- 如何验证新逻辑生效。
- 如何验证条件未命中时不影响原流程。
- 如何验证失败降级。
- 如何验证幂等。
- 如何验证兼容旧逻辑。
